# Prompt: Docs Sync

Read `GOAL.md` and inspect the implementation.

Bring documentation back into alignment with the code without turning the goal doc into a
low-level spec.

Good docs work for Verryte:

- clarify the finished shape of the engine
- explain current concepts in plain language
- document public commands or examples that actually exist
- keep input/control and agent-readiness clear
- mark aspirational behavior as aspirational when it is not implemented yet

Avoid:

- promising APIs that do not exist
- over-specifying internals too early
- burying the important input/control model

Update the smallest useful docs and report what changed.
